# Next-Level ResQNet

Advanced Real-Time Crisis Optimization System made by Viren Singh.

This is a static GitHub-ready research prototype. It runs directly from `index.html`
and does not need a backend or build step.

## What is real in this version

- Live public data ingestion from USGS earthquake GeoJSON feeds.
- Live public natural-event ingestion from NASA EONET v3 when available.
- Dijkstra shortest-path routing from response hubs to incidents.
- Edmonds-Karp max-flow allocation for scarce rescue capacity.
- Demand forecasting with a transparent, reproducible model.
- Measured outputs: coverage, unmet demand, bottlenecks, makespan, and browser runtime.

## How to run

Open `index.html` directly, or publish the folder with GitHub Pages.

## Important note

This is an educational and research prototype. It is not an official emergency
management system and should not be used for real rescue operations without
validation against verified operational data.
