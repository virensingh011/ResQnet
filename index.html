What makes this scientifically deeper?

Every displayed rescue number is derived from inspectable code. The app builds a graph, computes shortest paths, solves a max-flow allocation problem, measures unmet demand, and produces a reproducible forecast. Live data is used when public APIs are reachable; deterministic fallback fixtures keep the project runnable during offline review.

Dijkstra shortest paths Finds fastest travel-time routes from each response hub to each crisis zone.
Edmonds-Karp max flow Allocates scarce teams, ambulances, beds, and kits under edge-capacity constraints.
Demand forecasting Predicts near-future rescue demand before peak pressure using transparent features.
Auditable metrics Reports coverage, makespan, bottleneck edges, unmet need, and runtime.
